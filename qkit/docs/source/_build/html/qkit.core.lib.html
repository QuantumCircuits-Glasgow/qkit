query(*A, **K)[source]

Query the value for attribute. Will always return a single item if only one matches, otherwise it will return a list.

Tip: If you ALWAYS want a list, do:

list(DB.query(…))

Usage

Any combination of the following will works

Keywords: Only check equality

>>> DB.query(attrib=val)               
>>> DB.query(attrib1=val1,attrib2=val2)  # Match both

>>> DB.query({'attrib':val})                  
>>> DB.query({'attrib1':val1,'attrib2':val2}) # Match Both

Query Objects (DB.Q, DB.Qobj)

>>> Q = DB.Q() # Load Qobj from list_dict_DB
>>> DB.query(DQ.attrib == val)            
>>> DB.query( (Q.attrib1 == val1) &  (Q.attrib1 == val2) )  # Parentheses are important!
>>> DB.query( (Q.attrib1 == val1) &  (Q.attrib1 != val2))

update(*args, **queryKWs)[source]

Update an entry without needing to reindex the DB (or a specific attribute)

Usage:

>>> DB.update(updated_dict, query_dict_or_Qobj, query_attrib1=val1,...)
>>> DB.update(updated_dict, query_attrib1=val1,...)

Inputs:

updated_dictDictionary with which to update the entry. This is

done using the typical dict().update() construct to overwrite it

query_dict_or_Qobj
: Either the dictionary used in the query or a Qobj that

defines a more advanced query

query_attrib1=val1

: Additional (or sole) query attributes

DB Options:

This is set at instantiation but can be changed ‘DB’ is the list_dict_DB

DB.allowMultipleEdit

  • If True, will update multiple items

Notes:

  • Updating an item requires a deletion in a list that has length n equal to the number of items matching an attribute. This is O(n). However changing the entry directly and reindexing is O(N) where N is the size of the DB. If many items are changing and you do not need to query them in between, it may be faster to directly update the item and reindex
